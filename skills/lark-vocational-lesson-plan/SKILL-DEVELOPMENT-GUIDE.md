# AI+教育 Skill 开发规范指南

## 一、概述

本规范适用于 AI+教育领域的 Skill 开发，包括但不限于：
- 教案生成
- 授课计划编制
- 教学规范制定
- 课程标准设计
- 试卷生成
- 教学评价
- 其他教育相关场景

---

## 二、Skill 开发标准

### 2.1 SKILL.md 核心结构

```markdown
---
name: skill-name
description: "描述（100-150字符）"
---

# 标题

## 概述
## 核心工作流程
### 0. 模式选择
### 0.1 形式确认
### 0.2 信息收集
### 1. 输入处理与信息抽取
### 2. 资源加载与模板应用
### 3. 内容生成
### 4. 输出与质量控制
## 格式规范
## 使用示例
## 注意事项
```

### 2.2 目录结构规范

```
skill-name/
├── SKILL.md                    # 必需：入口文件
├── CHANGELOG.md               # 更新日志
├── references/
│   ├── templates/             # 模板文件
│   ├── examples/              # 示例文件
│   └── standards/             # 规范文件
└── assets/
    └── templates/            # 资源模板
```

## 三、核心设计原则

### 3.1 渐进式询问原则

**原则**：信息收集由浅入深，避免一次性要求用户提供全部信息

```
步骤1：收集基础信息（必需）
   ↓
步骤2：确认输出形式（必需）
   ↓
步骤3：收集补充材料（可选）
   ↓
步骤4：缺失兜底（大模型自动补充）
```

**示例**：

| 场景 | 步骤1 | 步骤2 | 步骤3 |
|------|-------|-------|-------|
| 教案生成 | 课程+课题 | 教案形式 | 材料上传 |
| 授课计划 | 专业+课程 | 计划格式 | 教学大纲 |
| 教学规范 | 规范类型 | 适用范围 | 行业标准 |

### 3.2 质量控制原则

**原则**：输出内容必须符合预设格式规范，不得随意发挥

| 控制点 | 要求 |
|--------|------|
| 编号规范 | 严格遵循层级编号规则 |
| 格式统一 | 与模板格式完全一致 |
| 内容完整 | 包含所有必需要素 |
| 质量门槛 | 内容符合专业标准 |

## 四、工作流程设计

### 4.1 标准工作流程

```
┌─────────────────────────────────────────┐
│           用户输入/触发 Skill           │
└─────────────────┬─────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│         步骤0：模式/形式选择             │
│    （如：精准/粗犷，表格式/讲义式）     │
└─────────────────┬─────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│         步骤1：基础信息收集             │
│    （课程、课题、专业、班级等）          │
└─────────────────┬─────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│         步骤2：材料/标准上传            │
│    （人才培养方案、课程标准等）          │
└─────────────────┬─────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│         步骤3：信息抽取与映射           │
│    （从材料中提取关键信息）             │
└─────────────────┬─────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│         步骤4：内容生成                 │
│    （按模板格式生成内容）               │
└─────────────────┬─────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│         步骤5：质量校验                 │
│    （格式检查、内容完整性）             │
└─────────────────┬─────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│         步骤6：输出与调整               │
│    （输出结果，支持用户调整）           │
└─────────────────────────────────────────┘
```

### 4.2 材料缺失处理机制

| 场景 | 处理方式 |
|------|----------|
| 有完整材料 | 优先使用材料内容，精准提取 |
| 部分材料缺失 | 大模型基于通用知识补充 |
| 无材料 | 全部由大模型基于通用知识生成 |

## 五、信息抽取映射

### 5.1 设计要求

每个 Skill 必须包含**信息抽取映射表**，明确：
- 输出的每个字段
- 对应的材料来源
- 提取的信息内容

### 5.2 示例模板

```markdown
### 信息抽取映射表

| 输出字段 | 材料来源 | 提取信息 |
|----------|---------|----------|
| 字段1 | 材料A | 信息1 |
| 字段2 | 材料B | 信息2 |
| ... | ... | ... |
```

## 六、格式规范

### 6.1 编号规则

| 层级 | 格式 | 示例 |
|------|------|------|
| 主标题 | 中文数字 | 一、二、三 |
| 子标题 | 阿拉伯数字 | 1. 2. 3. |
| 列表项 | 短横线 | - 项目 |
| 禁止 | 中文括号混用 | （一）（二） |

### 6.2 内容格式

- 教学分析：按模板结构分段
- 教学目标：统一使用 - 列表格式
- 教学过程：只保留统一表格，不展开详细说明

---

## 七、质量控制

### 7.1 生成前校验

- [ ] 必填信息是否完整
- [ ] 输出形式是否确认
- [ ] 材料是否上传

### 7.2 生成中控制

- [ ] 严格按模板格式生成
- [ ] 编号格式是否正确
- [ ] 内容是否完整

### 7.3 生成后检查

- [ ] 格式是否符合规范
- [ ] 内容是否完整
- [ ] 是否包含所有必需要素

---

## 八、测试与优化

### 8.1 测试场景

| 测试类型 | 场景 | 验证点 |
|----------|------|--------|
| 精准模式 | 完整材料 | 流程完整、信息准确 |
| 粗犷模式 | 基础信息 | 快速生成、格式正确 |
| 材料缺失 | 部分/无材料 | 兜底机制正常 |

### 8.2 用户反馈处理

- 收集格式问题 → 更新格式规范
- 收集内容问题 → 更新模板/提示
- 收集流程问题 → 优化工作流程

## 十、注意事项

| 原则 | 说明 |
|------|------|
| 渐进式询问 | 不一次性要求全部信息 |
| 格式严格 | 生成内容必须符合规范 |
| 质量把控 | 输出前必须校验 |
| 简化前确认 | 修改内容必须先与用户确认 |
| 版本管理 | 每次更新记录变更 |
