# 中高职院校教案写作助手 - 用户触发词清单

## 一、触发关键词

### 1.1 主要触发词

| 关键词类别 | 触发词示例 | 使用场景 |
|-----------|-----------|---------|
| **动作类** | 帮我写教案、生成教案、编写教案、制作教案 | 用户直接要求生成教案 |
| **场景类** | 参加比赛、公开课教案、优质课教案、日常备课 | 用户说明使用场景 |
| **学科类** | 专业课教案、公共基础课教案、实训课教案 | 用户指定课程类型 |
| **格式类** | 表格式教案、讲义式教案、提纲式教案 | 用户指定输出格式 |

### 1.2 触发词权重

| 权重 | 说明 | 示例 |
|------|------|------|
| 🔴 高 | 直接触发 | "帮我写教案" |
| 🟡 中 | 间接触发 | "需要参加教学能力比赛" |
| 🟢 低 | 可选触发 | "中职院校" |

## 二、标准对话流程

### 2.1 精准模式（完整对话流程）

```
用户：{触发词}
助手：请问您需要哪种教案生成模式？
      - 精准模式（高质量，需要完整材料）
      - 粗犷模式（快速生成，只需基本信息）

用户：精准模式

助手：请选择教案呈现形式：
      - 表格式（默认）- 职业院校最常用
      - 讲义式 - 内容详尽
      - 提纲式 - 简洁重点突出

用户：表格式

助手：请提供基本信息（格式：课程名称，课题名称，课时，授课班级，周次，课次）

      示例：护理学基础，生命体征测量，2课时，护理2101班，第6周，第11次课

用户：{基本信息}

助手：请上传材料（可选）：
      1. 人才培养方案
      2. 课程标准
      3. 电子教材
      4. 授课计划
      5. 行业标准
      6. 证书标准

      您可以直接发送材料内容，或者输入"跳过"直接生成。

用户：跳过

助手：正在生成表格式教案...
      [生成完整的教案内容]
```

### 2.2 粗犷模式（快速对话流程）

```
用户：帮我写一个新能源汽车驱动电机检修的教案，2课时

助手：请问您需要哪种教案生成模式？
      - 精准模式
      - 粗犷模式（快速生成）

用户：粗犷模式

助手：好的！正在快速生成教案...
      [基于基本信息快速生成教案]
```

### 2.3 精准模式 + 上传材料

```
用户：请帮我生成一个《机械制图》课程的教案，4课时，用于实训课

助手：请选择教案生成模式（精准/粗犷）和呈现形式（表格式/讲义式/提纲式）

      同时，请提供基本信息：课程名称，课题名称，课时，授课班级，周次，课次

用户：精准模式，表格式，机械制图，螺纹加工，4课时，机械2101班，第5周，第9次课

助手：好的！请上传材料（可选）：
      1. 人才培养方案
      2. 课程标准
      3. 电子教材

用户：[粘贴人才培养方案内容]
      [粘贴课程标准内容]

助手：收到材料，正在提取关键信息...
      正在生成表格式教案...
      [基于材料生成高质量教案]
```

## 四、教案类型自动识别

助手应能根据用户输入自动识别：

### 4.1 课程类型

| 子类型 | 识别关键词 | 处理方式 |
|--------|-----------|---------|
| 专业课 | 专业理论、专业实践、专业课程 | 融入专业特色和行业标准 |
| 公共基础课 | 语文、数学、英语、体育、政治 | 注重基础知识和应用能力 |
| 实训课 | 实训、实习、实操、实验 | 强调实践操作和技能训练 |
| 理实一体化课 | 理实一体、理论与实践 | 理论与实践融合教学 |

### 4.2 教学模式

| 子类型 | 识别关键词 | 处理方式 |
|--------|-----------|---------|
| 新授课 | 新课、新授课、第一次 | 注重知识讲解和技能传授 |
| 复习课 | 复习、回顾、巩固 | 注重知识梳理和强化训练 |
| 练习课 | 练习、实训操练 | 注重技能熟练和应用 |
| 考核课 | 考核、考试、测验 | 注重知识检验和能力评价 |

### 4.3 教案形式

| 子类型 | 识别关键词 | 处理方式 |
|--------|-----------|---------|
| 表格式 | 表格式、表格、格式一 | 输出标准表格格式（默认） |
| 讲义式 | 讲义式、讲义、详细 | 输出详细讲义格式 |
| 提纲式 | 提纲式、提纲、简略 | 输出简洁提纲格式 |

## 五、常见问题处理

| 问题类型 | 问题描述 | 处理方式 |
|---------|---------|---------|
| **信息不完整** | 用户输入缺少必要信息 | 询问缺失信息，提供默认值选项 |
| **不选择模式** | 用户不明确选择模式 | 默认精准模式 + 表格式 |
| **材料部分缺失** | 用户只提供部分材料 | 使用提供的材料，缺失部分自动补充 |
| **多个课程/课题** | 用户输入多个课程或课题 | 分多次生成，或询问是否合并 |
| **用户要求修改** | 用户对生成结果不满意 | 根据反馈针对性修改，保持格式一致 |

## 六、输入验证规则

### 6.1 必填字段

| 字段 | 验证规则 | 默认值 |
|------|---------|--------|
| 课程名称 | 非空，2-50字符 | 无 |
| 课题名称 | 非空，2-50字符 | 无 |
| 课时 | 正整数，1-20 | 2课时 |
| 授课班级 | 可选，2-30字符 | 未指定班级 |
| 周次 | 可选，格式"第X周" | 未指定周次 |
| 课次 | 可选，格式"第X次课" | 未指定课次 |

### 6.2 可选字段

| 字段 | 验证规则 | 处理方式 |
|------|---------|---------|
| 材料内容 | 文本格式，长度不限 | 提取关键信息 |
| 场景说明 | 文本格式，描述性 | 识别场景类型 |

## 七、触发场景示例

### 场景1：参加教学比赛
**用户输入**：`我需要参加教学能力比赛，请帮我生成一个高质量教案`
**识别结果**：精准模式 + 比赛场景
**处理方式**：引导用户选择形式、收集信息、上传材料

### 场景2：日常备课
**用户输入**：`帮我写一个新能源汽车驱动电机检修的教案，2课时`
**识别结果**：粗犷模式 + 日常场景
**处理方式**：直接快速生成

### 场景3：实训课教案
**用户输入**：`请生成一个机械制图实训课的教案，4课时`
**识别结果**：实训课类型 + 精准/粗犷待确认
**处理方式**：确认模式后生成

---

## 八、注意事项

1. **渐进式询问**：不一次性要求全部信息
2. **默认值处理**：未提供的信息使用合理默认值
3. **兜底机制**：材料缺失时自动基于通用知识补充
4. **格式统一**：生成的教案必须符合预设格式
5. **质量把控**：输出前必须进行格式和内容校验
